# Анализ фронтенда и обучающего контента (Qwik City MVP)

## 1. Архитектура фронтенда

Проект реализован как **Web MVP** с использованием современного стека, ориентированного на высокую производительность (особенно на мобильных устройствах).

### Выбранный стек технологий:
- **Фреймворк:** Qwik + Qwik City
- **Язык:** TypeScript
- **Стилизация:** Qwik Scoped CSS (`useStylesScoped$` и `*.css?inline`)
- **Тестирование:** Vitest (unit/component) и Playwright (e2e)

**Преимущества стека:**
Использование Qwik дает возможность добиться быстрой начальной загрузки (first load) за счет "resumability" — отправки минимального количества JavaScript на клиент. Фреймворк `Qwik City` позволяет использовать file-based routing и объединять SSR, загрузку данных (`routeLoader$`) и обработку действий (`routeAction$`) в одном приложении без необходимости поднятия отдельного backend-сервера (вместо него используется паттерн BFF - Backend-for-Frontend).

### Архитектурный стиль
Кодовая база организована по принципам **Feature-Sliced Design (FSD)** с "тонкими" роутами:
- **`routes/`**: Содержат только логику оркестрации экрана, `routeLoader$` и `routeAction$`. Здесь нет бизнес-логики.
- **`features/`**: Уровень пользовательских сценариев (use-cases), таких как лента (`feed`), интерактивные элементы (`interactions`), геймификация (`gamification`).
- **`entities/`**: Доменные сущности (карточки, прогресс пользователя, стрик).
- **`shared/`**: Инфраструктурный слой (API клиенты, аналитика, конфигурации, базовые UI-компоненты).

*Строгие правила зависимостей:*
Роуты могут импортировать фичи, сущности и shared; фичи — сущности и shared; сущности — только shared. Инверсия запрещена.

### Ключевые механизмы фронтенда
- **Единый рендерер:** Все обучающие карточки рендерятся через единый компонент `CardRenderer.tsx`. Поддерживаются различные типы карточек (`video`, `carousel`, `quiz`, `match`, `mini_game`, `recap`).
- **Анти-думскролл:** В `features/feed/model/anti-doomscroll.ts` реализована логика, предотвращающая пассивный скроллинг:
  - Нельзя показывать более двух (`MAX_SAME_TYPE_STREAK = 2`) карточек одного типа подряд.
  - Каждые 4 карточки (`RECAP_INTERVAL = 4`) вставляется карточка-рекап (Recap) для закрепления материала.
- **Интернационализация (i18n):** Поддерживается выбор языка (en/es) через куки (`LEARNING_LANGUAGE_COOKIE`), что влияет на выдачу контента.

## 2. Архитектура обучающего контента

Обучающий контент организован в виде модульной системы (домен `src/slides/`). В настоящее время контент доступен для уровней от `A1`, с возможностью расширения до `C1`.

### Структура контента
Каждый модуль описывает определенную тему и состоит из:
- **Метаданных:** Уровень (например, `a1`), категория (`grammar`, `vocabulary`, `listening`, `reading`, `speaking`), теги правил (rule tags).
- **Карточек (`Card[]`):** Содержат заголовок, описание, данные для интерактивного взаимодействия (`interaction`) и награду за прохождение (`xp`).

Контент разделен по языкам:
- **Английский (en):** Содержит модули (например, грамматика, лексика).
- **Испанский (es):** Имеет богатую структуру в `modules/es/a1/`, включая:
  - Грамматика: `ser-estar`, `presente-basico`, `articulos-genero-numero` и генерация карточек из CSV (`csv-flashcards.module.ts`).
  - Лексика: `familia-comida`, `numeros-0-100`, `viajes`.
  - Аудирование, чтение и говорение.

Файл `flashcards.csv` содержит пары вопрос-ответ для генерации quiz-карточек (в основном испанский язык).

### Механизм выдачи контента (Engine)
Файл `src/slides/engine/select-slides.ts` отвечает за формирование ленты:
1. **Фильтрация:** Модули отбираются на основе профиля пользователя (целевой уровень, предпочитаемые категории).
2. **Скоринг (Ранжирование):** Каждому модулю присваиваются баллы:
   - Совпадение уровня: +6 баллов
   - Совпадение категории: +4 балла
   - Совпадение фокусных правил (`focusRules` профиля): +2 балла за каждое совпадение.
3. **Интерливинг (Перемешивание):** Карточки из отсортированных модулей извлекаются поочередно (interleaving), чтобы в ленте чередовались разные темы (например, грамматика с лексикой), обеспечивая разнообразие.
4. **Пагинация:** Карточки возвращаются порциями на основе `cursor` и `limit`.

### Геймификация
Каждая карточка имеет поле `reward`, определяющее количество получаемых `xp` за её прохождение. Платформа также отслеживает ежедневные стрики (streak) и ежедневные цели (daily goal), мотивируя пользователя возвращаться в приложение.

## Итог
Архитектура фронтенда и контента плотно интегрированы: фронтенд обеспечивает быструю доставку контента и соблюдение правил "здорового" потребления информации (анти-думскролл), а движок контента динамически подстраивает выдачу карточек под уровень и интересы пользователя, чередуя темы для максимальной эффективности обучения.
