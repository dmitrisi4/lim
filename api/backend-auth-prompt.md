# Задача на бекенд: Интеграция Google OAuth2 Авторизации

Привет! Нам нужно добавить авторизацию через Google для нашего проекта (Lim). Фронтенд на Qwik уже подготовлен к этому.

## Требования к реализации на бекенде:

1. **Библиотеки/Подходы**:
   - Используйте OAuth2 (например, через `passport-google-oauth20`, если это Node.js).
   - Для хранения сессии используйте JWT, который должен передаваться в **HttpOnly** куках для безопасности.

2. **Эндпоинты (Endpoints)**:
   - `GET /auth/google` — Этот роут должен инициировать OAuth-флоу, перенаправляя пользователя на страницу входа Google. На него фронтенд переходит при клике на кнопку "Sign In with Google".
   - `GET /auth/google/callback` — Колбэк от Google, который принимает `code`, обменивает его на токен доступа Google и получает профиль пользователя.
     - **Логика колбэка**: 
       1. Найти или создать пользователя в базе данных (используя `googleId` или `email`).
       2. Сгенерировать JWT для сессии.
       3. Ответить заголовком `Set-Cookie` для установки HttpOnly, Secure (в production) куки с JWT (с атрибутом `SameSite=Lax` или `None`, в зависимости от того, находятся ли фронт и бекенд на одном домене).
       4. Сделать редирект на фронтенд (например, на `http://localhost:5173/` или другой URL через переменные окружения).
   - `GET /auth/me` — Роут для получения текущего пользователя фронтендом.
     - Должен проверять HttpOnly куку с JWT.
     - Возвращать объект: `{ "id": "...", "email": "...", "name": "...", "avatarUrl": "..." }` или ошибку 401, если сессия истекла/отсутствует.

3. **CORS и Cookies**:
   - Фронтенд отправляет запросы с параметром `credentials: "include"`, поэтому настройка CORS на бекенде должна разрешать источник фронтенда (`origin`) и иметь `credentials: true`. Это критично для работы кук между разными портами на localhost.

4. **Переменные окружения**, которые понадобятся бекенду:
   - `GOOGLE_CLIENT_ID`
   - `GOOGLE_CLIENT_SECRET`
   - `GOOGLE_CALLBACK_URL`
   - `JWT_SECRET`
   - `FRONTEND_URL` (для финального редиректа после успешной авторизации, например `http://localhost:5173`)

## Пример User сущности на фронтенде:
```typescript
export type UserType = {
	id: string;
	email: string;
	name: string;
	avatarUrl?: string; // (опционально) картинка из профиля Google
};
```
Ожидаем, что объект, возвращаемый из роута `/auth/me`, будет иметь аналогичные поля.
